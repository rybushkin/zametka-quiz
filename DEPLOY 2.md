# Деплой Zametka на Vercel и свой домен

## Предусловия

- Код в Git (GitHub / GitLab / Bitbucket).
- Аккаунт на [Vercel](https://vercel.com).

## Шаги

### 1. Подключить репозиторий к Vercel

1. Зайди в [vercel.com](https://vercel.com) → **Add New** → **Project**.
2. Импортируй репозиторий с проектом Zametka.
3. **Важно:** в настройках проекта задай **Root Directory**: `web` (корень приложения — там лежит `package.json` и Next.js).
4. **Build Command** и **Output Directory** можно не менять (Vercel подхватит Next.js сам).
5. Нажми **Deploy**.

После первого деплоя сайт будет доступен по адресу вида `https://<project>.vercel.app`.

### 2. Подключить свой домен

1. В проекте на Vercel: **Settings** → **Domains**.
2. Нажми **Add** и введи свой домен (например `zametka.example.com` или `example.com`).
3. Vercel покажет, какие DNS-записи нужно добавить у регистратора:
   - для поддомена обычно: **CNAME** `zametka` → `cname.vercel-dns.com`;
   - для корня домена — **A** на IP Vercel (инструкция будет в интерфейсе).
4. Сохрани записи у хостинга DNS, подожди распространения (до 24–48 ч, часто быстрее).
5. В Vercel нажми **Verify** — после успешной проверки домен будет обслуживаться Vercel (HTTPS включится автоматически).

### 3. Проверка

- Открой свой домен в браузере — должна открываться квиз-страница Zametka.
- Убедись, что картинки из `public/` (например `/img/cover.png`) загружаются.

## Откат

- В Vercel: **Deployments** → выбери предыдущий деплой → **Promote to Production**.
- Домен менять не нужно — он указывает на тот же проект.

## Заметки

- Секреты и env-переменные (если появятся) задаются в **Settings** → **Environment Variables**.
- При пуше в ветку по умолчанию Vercel делает прод-деплой; превью-деплои создаются для других веток и PR.
